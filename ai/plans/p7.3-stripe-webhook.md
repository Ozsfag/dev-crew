# П7.3 — Stripe Webhook реализация

## Контекст

`StripeWebhookAdapter` существует как заглушка с тремя `TODO`-комментариями.
При получении события `customer.subscription.created` или `customer.subscription.deleted`
нужно обновлять план организации (FREE ↔ PRO). Без реализации биллинг не работает —
пользователи платят, но план не меняется.

## Проблема

`billing/adapter/in/stripe/StripeWebhookAdapter.java` содержит:
- `TODO: Stripe SDK — Webhook.constructEvent()` — нет проверки подписи (security risk)
- `TODO: customer.subscription.created → апгрейд плана до PRO` — не реализовано
- `TODO: customer.subscription.deleted → даунгрейд до FREE` — не реализовано

Зависимость: нужна связь billing → organization для обновления `OrgPlan`.

## Техническое решение

### 3. StripeWebhookPort — порт для обновления плана

```java
// billing/domain/StripeWebhookPort.java
public interface StripeWebhookPort {
  void handleSubscriptionCreated(String stripeCustomerId);
  void handleSubscriptionDeleted(String stripeCustomerId);
}
```

### 4. StripeWebhookService — реализация

```java
// billing/app/service/command/StripeWebhookService.java
@Service
@RequiredArgsConstructor
public class StripeWebhookService implements StripeWebhookPort {

  private final OrganizationCommandService organizationCommandService;

  @Override
  @Transactional
  public void handleSubscriptionCreated(String stripeCustomerId) {
    organizationCommandService.updatePlanByStripeCustomer(stripeCustomerId, OrgPlan.PRO);
  }

  @Override
  @Transactional
  public void handleSubscriptionDeleted(String stripeCustomerId) {
    organizationCommandService.updatePlanByStripeCustomer(stripeCustomerId, OrgPlan.FREE);
  }
}
```

### 5. OrganizationCommandService — добавить метод

```java
// organization/app/service/command/OrganizationCommandService.java — добавить:
public void updatePlanByStripeCustomer(String stripeCustomerId, OrgPlan newPlan) {
  var org = organizationStore.findByStripeCustomerId(stripeCustomerId)
      .orElseThrow(() -> new NotFoundException("Organization", stripeCustomerId));
  var updated = new OrganizationModel(org.id(), org.name(), newPlan,
      org.createdAt(), timeProvider.now());
  organizationStore.save(updated);
}
```

### 6. OrganizationModel — добавить stripeCustomerId

```java

    UUID id, String name, OrgPlan plan,
```

### 7. StripeWebhookAdapter — заменить заглушку

```java
// billing/adapter/in/stripe/StripeWebhookAdapter.java
@Slf4j
@RestController
@RequestMapping("/api/stripe")
@RequiredArgsConstructor
public class StripeWebhookAdapter {

  private final StripeWebhookPort stripeWebhookPort;
  private final BillingProperties billingProperties;

  @PostMapping("/webhook")
  public ResponseEntity<Void> handleWebhook(
      @RequestBody String payload,
      @RequestHeader("Stripe-Signature") String signature) {
    Event event;
    try {
      event = Webhook.constructEvent(payload, signature, billingProperties.getStripeWebhookSecret());
    } catch (SignatureVerificationException e) {
      log.warn("Невалидная подпись Stripe webhook");
      return ResponseEntity.badRequest().build();
    }

    switch (event.getType()) {
      case "customer.subscription.created" -> {
        var subscription = (Subscription) event.getDataObjectDeserializer()
            .getObject().orElseThrow();
        stripeWebhookPort.handleSubscriptionCreated(subscription.getCustomer());
      }
      case "customer.subscription.deleted" -> {
        var subscription = (Subscription) event.getDataObjectDeserializer()
            .getObject().orElseThrow();
        stripeWebhookPort.handleSubscriptionDeleted(subscription.getCustomer());
      }
      default -> log.debug("Игнорируем событие Stripe: type={}", event.getType());
    }
    return ResponseEntity.ok().build();
  }
}
```

### 8. StripeWebhookAdapterTest

```java
@ExtendWith(MockitoExtension.class)
class StripeWebhookAdapterTest {

  @Mock private StripeWebhookPort stripeWebhookPort;
  private MockMvc mockMvc;

  @BeforeEach
  void setUp() {
    var properties = new BillingProperties();
    properties.setStripeWebhookSecret("whsec_test");
    var adapter = new StripeWebhookAdapter(stripeWebhookPort, properties);
    mockMvc = MockMvcBuilders.standaloneSetup(adapter)
        .setControllerAdvice(new GlobalExceptionHandler())
        .build();
  }

  @Test
  void POST_webhook_returns_400_for_invalid_signature() throws Exception {
    mockMvc.perform(post("/api/stripe/webhook")
            .contentType(MediaType.TEXT_PLAIN)
            .header("Stripe-Signature", "invalid")
            .content("{}"))
        .andExpect(status().isBadRequest());
    verifyNoInteractions(stripeWebhookPort);
  }

  // Тест для subscription.created и subscription.deleted — через Stripe mock или
  // Webhook.constructEvent с тестовым ключом (whsec_test):
  // см. Stripe testing docs: https://stripe.com/docs/webhooks/test
}
```
