# П10.6 — Инфраструктура: Docker и CI/CD

## Контекст

Dockerfile использует JRE (не может запускать Gradle), CI не запускает интеграционные тесты,
docker-compose содержит hardcoded путь, нет health indicator для LLM.

---

## 6.1 Dockerfile: JRE не может запускать Gradle тесты

**Файл**: `docker/Dockerfile`

**Проблема**: runtime image — `eclipse-temurin:21-jre-alpine`. JRE не содержит `javac`.
Агенты (`GradleTools.runTests()`) не смогут запускать тесты.

**Решение**: переключить на `21-jdk-alpine` (+80 MB):
```dockerfile
FROM eclipse-temurin:21-jdk-alpine AS runtime
```

**Acceptance Criteria**:
- [x] Dockerfile использует `eclipse-temurin:21-jdk-alpine` для runtime
- [x] `docker build` проходит успешно
- [x] Образ < 600 MB

---

## 6.2 CI: нет шага интеграционных тестов

**Файл**: `.github/workflows/ci.yml`

**Проблема**: CI запускает только unit тесты. Интеграционные тесты с Testcontainers не запускаются.

**Решение**: добавить job `integration-tests`:
```yaml
integration-tests:
  runs-on: ubuntu-latest
  needs: test
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-java@v4
      with:
        java-version: '21'
        distribution: 'temurin'
    - run: ./gradlew test -Dspring.profiles.active=tc
```

**Acceptance Criteria**:
- [x] Integration tests job в `ci.yml`
- [x] Job запускается после unit tests (needs: test)
- [x] Failure в integration tests блокирует merge

## 6.3 docker-compose.yml: hardcoded local path

**Файл**: `docker/docker-compose.yml`

**Проблема**: `${PROJECTS_ROOT:-/Users/artemsergienko/IdeaProjects}:/projects` —
хардкоден путь разработчика.

**Решение**: убрать hardcoded дефолт:
```yaml
volumes:
  - ${PROJECTS_ROOT}:/projects
```

Документировать в `.env.example`.

**Acceptance Criteria**:
- [x] Нет hardcoded username в `docker-compose.yml`
- [x] `docker/.env.example` содержит `PROJECTS_ROOT=`

## 6.4 Добавить health indicator для LLM

**Проблема**: `/actuator/health` не проверяет Anthropic API. Deployment считается healthy,
но LLM недоступен.

**Решение**:
```java
@Component
public class AnthropicHealthIndicator implements HealthIndicator {
  private final ChatLanguageModel model;

  @Override
  public Health health() {
    try {
      model.generate("ping");
      return Health.up().build();
    } catch (Exception e) {
      return Health.down().withDetail("error", e.getMessage()).build();
    }
  }
}
```

**Acceptance Criteria**:
- [x] `/actuator/health` включает `anthropic` indicator
- [x] При недоступном API ключе — `DOWN`
- [x] Не вызывается при выключенном `ANTHROPIC_API_KEY`
