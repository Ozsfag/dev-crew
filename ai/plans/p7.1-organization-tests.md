# П7.1 — Тесты для `organization/` контекста

## Контекст

CLAUDE.md требует 100% покрытия кода. Контекст `organization/` полностью реализован
(сервисы, контроллер, JPA-слой), но не имеет ни одного unit-теста. При рефакторинге
или добавлении нового поля — никакой страховки.

## Проблема

Непокрытые классы:
- `organization/app/service/command/OrganizationCommandService.java` — ~4 публичных метода
- `organization/app/service/query/OrganizationQueryService.java` — ~3 публичных метода
- `organization/adapter/in/web/controller/OrganizationController.java` — ~5 endpoint-ов

Все ветки `if`/`else`, все `throw NotFoundException` — без тестов.

### OrganizationControllerTest

Standalone MockMvc без `@SpringBootTest`:

```java
@ExtendWith(MockitoExtension.class)
class OrganizationControllerTest {

  @Mock private OrganizationCommandService commandService;
  @Mock private OrganizationQueryService queryService;
  private MockMvc mockMvc;

  @BeforeEach
  void setUp() {
    var controller = new OrganizationController(
        commandService, queryService,
        Mappers.getMapper(OrganizationWebMapper.class));
    mockMvc = MockMvcBuilders.standaloneSetup(controller)
        .setControllerAdvice(new GlobalExceptionHandler())
        .build();
  }

  @Test void POST_organizations_returns_201() throws Exception { ... }
  @Test void GET_organizations_id_returns_404_when_missing() throws Exception { ... }
  @Test void POST_organizations_returns_400_when_name_missing() throws Exception { ... }
}
```
